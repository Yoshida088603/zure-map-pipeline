# 公図と現況のずれ — 凡例（ポリゴン色分け）

国土地理院系の資料で用いられる**ずれ量の区分**と、本リポジトリでの対応関係。

## 属性の意味（重要）

| 属性 | 型の例 | 意味 |
|------|--------|------|
| **`BFR_GOSA`**（ずれ・前） | 実数（m） | **公図と現況のずれの大きさ（メートル）**。凡例表の「10cm未満」等はこの値の閾値で切る。 |
| **`BFR_RANK`**（ランク・前） | 整数 **1〜5** | **上記ずれ量に対応した区分コード**（帯ごとにまとまる）。練馬区サンプルでは RANK=1 のとき GOSA∈[0,0.094]、RANK=2 で [0.12,0.3] … のように凡例と整合。 |
| **`AFT_GOSA` / `AFT_RANK`** | 同上 | 処理・表示後（ずれ・後／ランク・後）。**前と後は常に同じとは限らない**（同一市区町村内でも GOSA が一致しない件あり）。地図の公式系凡例に合わせるなら通常は **ずれ(前) `BFR_GOSA`** を使う。 |

**誤り:** `BFR_RANK - BFR_GOSA` をメートルとして `step` に渡すこと。`RANK` は最大 5 程度の区分コードなので、例えば `4 - 0.2 = 3.8` と大きく見え、黄・緑帯が **薄ピンク帯（1〜10m）** に落ちる。

## 色分けに使う値

地図の塗り分け入力 \(z\) は **メートル単位の `BFR_GOSA`**（0 未満はデータ異常扱いで別色）。

## 区分表

MapLibre の `step` は **\[stop_i, stop_{i+1})** で色が切り替わる。**入力が 0.1 未満**に既定色（薄青）を対応させる。

| 分類 | 備考（ずれの大きさ） | 入力 \(z\)（m） | 代表色（HTML） |
|------|----------------------|-----------------|----------------|
| 精度が高い地域 | 10cm未満 | \(z < 0.1\) | 薄い青 `#87ceeb` |
| 小さなずれのある地域 | 10cm以上30cm未満 | \(0.1 \le z < 0.3\) | 薄い緑 `#98fb98` |
| ずれのある地域 | 30cm以上1m未満 | \(0.3 \le z < 1\) | 薄い黄 `#fffacd` |
| 大きなずれのある地域 | 1m以上10m未満 | \(1 \le z < 10\) | 薄いピンク `#ffb6c1` |
| きわめて大きなずれのある地域 | 10m以上 | \(z \ge 10\) | グレー `#9e9e9e` |

`BFR_GOSA` 欠落、または **負の値**（想定外）は **別色** `#bdbdbd`。

## MapLibre 式（参照実装）

`main.js` の `zureDeviationFillColor` と同一。

```js
[
  'case',
  ['any', ['!', ['has', 'BFR_GOSA']], ['<', ['to-number', ['get', 'BFR_GOSA']], 0]],
  '#bdbdbd',
  [
    'step',
    ['to-number', ['get', 'BFR_GOSA']],
    '#87ceeb',
    0.1, '#98fb98',
    0.3, '#fffacd',
    1, '#ffb6c1',
    10, '#9e9e9e',
  ],
]
```

## よくある誤り

- **`BFR_RANK - BFR_GOSA` で色分け** → 区分コードと長さを混同し、緑・黄がピンクに見える。
- **`step` の既定色を誤配置** → 10cm未満が灰色になる（古い実装の別バグ）。
